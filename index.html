プロジェクト概要

このプロジェクトは会議室予約システムを開発するものです。主な機能は以下の通りです。

主な機能
  • ログイン機能: ユーザーIDとパスワードでログインします。ユーザーが存在しない場合はエラーメッセージを表示します。
  • 予約画面:
    • 日付、時間帯、会議室を選択して予約できます。
    • 自分が予約した場所は水色、他者が予約した場所はグレーで表示されます。
    • ログイン日を初期表示とし、前日・翌日の予約状況も確認できます。
  • 予約・解除機能:
    • チェックを入れた部屋と時間帯の組み合わせで予約を登録・解除できます。
    • 予約情報は「洗い替え」方式で、以前の登録情報を論理削除し、新しい情報を新規登録します。
  • ログアウト機能: ログインセッションを削除し、ログイン画面へ戻ります。
使用技術
  • 開発言語: Java
  • フレームワーク: Java Spring Framework + JPA + Maven
  • データベース: MariaDB
テーブル設計

システムは4つのテーブルを使用します。

  • ユーザーテーブル（user）: ユーザーID（user_id）とパスワード（password）、ユーザー名（user_name）を管理します。
  • 予約情報テーブル（t_reservation）: 予約の主情報（ID、ユーザーID、予約日、部屋ID、時間帯IDなど）を管理します。is_deletedフラグで論理削除を扱います。
  • 会議室マスタテーブル（m_room）: 会議室名（name）と削除フラグ（is_deleted）を管理します。
  • 時間帯マスタテーブル（m_time）: 時間帯名（name）と削除フラグ（is_deleted）を管理します。

仕様概要

1. 技術スタック
  • 開発言語: Java/javascript
  • フレームワーク: Spring Framework (Spring Boot)
  • RDBMS: MariaDB
  • O/Rマッパー: Hibernate (spring-boot-starter-data-jpa)
  • ビルドツール: Maven
2. 機能要件
  • ログイン機能: ユーザーIDとパスワードによる認証。認証失敗時はエラーメッセージを表示。パスワードはBCryptでハッシュ化して保存。
  • メイン画面: ログイン後に掲示板一覧を表示。
  • 投稿機能: 投稿内容を入力し、データベースに登録。投稿内容は50文字以内の制限あり。
  • 削除機能: 自分の投稿のみ削除可能。ログインユーザーと投稿ユーザーが一致する場合にのみ削除ボタンが表示される。
  • ログアウト機能: セッションを破棄し、ログイン画面へ戻る。
3. 画面設計
  • ログイン画面: ユーザーIDとパスワードの入力フォーム、ログインボタン、エラーメッセージ表示エリア（IDまたはパスワードが間違っている場合）。
  • メイン画面: ログアウトボタン、投稿内容入力フォーム、投稿ボタン、投稿一覧表示エリア、および各投稿の削除ボタン（投稿者本人の場合）。
